# Custom themes

see·el·aye ships with three built-in themes:
- [Solarized dark](?path=/docs/containers-terminal--solarized-dark#solarized-dark)
- [Solarized light](?path=/docs/containers-terminal--solarized-light#solarized-light)
- [One dark](?path=/docs/containers-terminal--one-dark#one-dark)

While these are all very beautiful and functional themes, your tastes
[may be different](https://iterm2colorschemes.com), in which case see·el·aye
allows you to define your own themes.

## Guide

see·el·aye's color system is based on the standard 16-color ANSI system, albeit
with a few modifications.

### Color palette

To create a theme for see·el·aye, you need to define 21 colors.

- 8 normal colors
  - `--color-normal-black`
  - `--color-normal-red`
  - `--color-normal-green`
  - `--color-normal-yellow`
  - `--color-normal-blue`
  - `--color-normal-magenta`
  - `--color-normal-cyan`
  - `--color-normal-white`
- 8 bright colors
  - `--color-bright-black`
  - `--color-bright-red`
  - `--color-bright-green`
  - `--color-bright-yellow`
  - `--color-bright-blue`
  - `--color-bright-magenta`
  - `--color-bright-cyan`
  - `--color-bright-white`
- 5 content colors
  - `--color-normal-bg`
  - `--color-highlight-bg`
  - `--color-normal-fg`
  - `--color-highlight-fg`
  - `--color-secondary-fg`

Any CSS file that exposes a class defining these 21 CSS variables is a valid
theme file. A theme file can export more than one theme, such as that of
Solarized, which provides both a dark and a light theme from the same CSS file.

### Registration

Follow instructions upto the
[Render](?path=/story/guide-quickstart--page#render) section of the quickstart.

```html
<template>
  <Terminal class="my custom theme classes"/>
</template>

<!-- everything else remains the same -->

<style scoped lang="css" src="<path/to/my/theme>.css"></style>
```
